Syntax tutorial for Dark Pages

The Dark Pages tries to make writing articles as simple as possible given the restraints of the web technologies we try to rely on. Our web server is entirely made from rust/hyper and we rely heavily on minimising the bandwidth as much as we can

We have developed a schema for articles that is consistent, easy to understand, and compact. This article will explain how to write articles for the Dark Pages.

fn main() -> i32 { println!("Hello world!"); }

Article File Layout

So for each article, we have a dedicated folder with the directory name for the article (e.g "syntax-tutorial-17-oct-2023"). This name is only used for the url and internal usage.

Within the article folder, we have two files: data.toml and markup.html. data.toml is just a basic toml file that describes the article to the server program so it can list it. markup.html is the html code for the article's page

Here is an example of data.toml:

name = "Epic article for dark pages!"
author = "Your Name"
date = "unix timestamp (https://unixtime.org/)"
tags = "epic|article|something|woah"
